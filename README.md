## Working with Power Query M files

Some examples may include standalone `.pq`, `.pqm`, or `.m` files containing Power Query M code.

You can use these by:

1. Opening Power Query in Excel or Power BI.
2. Creating a blank query.
3. Opening the Advanced Editor.
4. Copying the M code into the editor.
5. Reviewing the steps and adjusting file paths or source data as needed.

Some queries may require you to update file paths before they work on your computer.

## Important note about file paths

Power Query examples that connect to local files may include file paths that are different from the paths on your computer.

For example, a query may refer to a file location such as:

```text
C:\Users\Melissa\Documents\Example Files\
```

You may need to change this path to match where you saved the files.

When possible, examples will be structured to make this easy to update.

## Learning approach

This repository is designed to help you do more than copy and paste a finished solution.

Power Query becomes much easier when you understand the logic behind each step. The examples in this repository are intended to help you see how a query is built, why certain choices are made, and how you can adapt the same ideas to your own data.

The focus is on building understanding through practical examples, including:

- how Power Query steps are created and connected
- what the M code behind those steps is doing
- how to break a problem into smaller, manageable pieces
- how to recognize patterns in messy data
- how to troubleshoot errors and unexpected results
- how to explore different ways to solve the same problem
- how to turn a one-time solution into something reusable

Some examples may show more than one approach. This is intentional. In Power Query, there is often more than one way to solve a problem, and comparing approaches can help you understand the strengths, trade-offs, and limitations of each solution.

The goal is to help you become more independent with Power Query, so you can move from following instructions to confidently building and modifying queries.
